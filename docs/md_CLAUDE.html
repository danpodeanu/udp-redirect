Project

UDP Redirect is a single-file, high-performance UDP packet redirector for scenarios where layer-level redirection is impractical (e.g., Wireguard VPN, DNS, Minecraft). Targets Linux x64 and macOS/Darwin ARM64. Licensed GPLv2.

Build

make # Build (gcc -Wall -O3)
make clean # Remove build artifacts and docs
make install # Install to PREFIX (default: /usr/local)
make docs # Generate Doxygen HTML documentation

Direct compilation alternative: gcc udp-redirect.c -o udp-redirect -Wall -O3

No external dependencies — only standard C library and POSIX APIs.

CI

GitHub Actions runs make on ubuntu-latest for pushes/PRs to main. No automated functional tests exist; the only test is udp-redirect --version in the Homebrew formula.

Architecture

Single source file: udp-redirect.c (~1050 lines). The design is intentionally single-file for auditability and deployment simplicity.

Core loop: Poll-based non-blocking I/O with two UDP sockets (listen and send). Poll timeout is 1000ms. Packets received on the listen socket are forwarded to the connect endpoint, and replies are forwarded back.

Key structures:

  • struct settings — all command-line configuration
  • struct statistics — packet/byte counters with 60-second windowed and cumulative totals

Platform conditionals: #ifdef __MACH__ for macOS (IP_BOUND_IF), #elif __unix__ for Linux (SO_BINDTODEVICE).

Conventions

  • DEBUG macro: DEBUG(debug_level, message_level, fmt, ...) — pass the debug level variable explicitly as the first argument, do not hardcode a variable name.
  • Naming: sockets use lsock/ssock, addresses use laddr/caddr/saddr, ports use lport/cport/sport.
  • Error handling: Harmless network errors (EAGAIN, EHOSTUNREACH, etc.) are ignored by default via ERRNO_IGNORE_* macros. Critical errors call exit(EXIT_FAILURE).
  • No dynamic allocation in the main loop — static buffers and stack-based structures only.
  • Commit style: Mix of conventional commit prefixes (style:, nit:) and plain descriptions. Lowercase after prefix.
